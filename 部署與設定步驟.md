# 社團學習平臺 — Oracle Cloud 部署與設定步驟

Moodle 自架，跑在 Oracle Cloud Always Free 的 Ubuntu ARM VM 上。
本資料夾已備好所有檔案，跟 VM 無關；只有**公網 IP／網域**在最後填一次。

---

## 前置（你本人先做）
- 開好 Oracle Always Free 的 **Ubuntu 22.04/24.04、Ampere A1** VM（2 OCPU / 12GB 起）
- 存好 SSH 私鑰
- Security List 開 **80、443**（預設只開 22）
- 記下 VM 的**公網 IP**

---

## 步驟一：連進 VM 並裝 Docker

```bash
ssh -i 你的私鑰.key ubuntu@<公網IP>

# 裝 Docker + Compose
sudo apt-get update
curl -fsSL https://get.docker.com | sudo sh
sudo usermod -aG docker $USER
newgrp docker    # 或登出重連，讓群組生效
```

> Oracle Ubuntu 映像預設有 iptables 擋 80/443，除了雲端 Security List 也要放行主機防火牆：
> ```bash
> sudo iptables -I INPUT 6 -m state --state NEW -p tcp --dport 80 -j ACCEPT
> sudo iptables -I INPUT 6 -m state --state NEW -p tcp --dport 443 -j ACCEPT
> sudo netfilter-persistent save
> ```

## 步驟二：把這個資料夾傳上去

在 VM 上：
```bash
mkdir ~/club_lms && cd ~/club_lms
```
把本機 `club_lms\` 內所有檔案傳進去（scp 或 VSCode remote 皆可）。例如本機 PowerShell：
```powershell
scp -i 你的私鑰.key -r C:\Users\Alber\Claude\club_lms\* ubuntu@<公網IP>:~/club_lms/
```

## 步驟三：設定與啟動

```bash
cd ~/club_lms
cp .env.example .env
nano .env                 # 改掉兩個密碼

sed -i 's/\r$//' setup.sh # 保險：去掉 Windows 換行
chmod +x setup.sh
./setup.sh                # 下載 Moodle+外掛、建置、啟動、設權限
```

## 步驟四：跑安裝精靈（唯一要填 IP 的地方）

瀏覽器打開 `http://<公網IP>`，照精靈走：
- 語言：正體中文
- 網站位址(wwwroot)：`http://<公網IP>`（之後接網域再改，見文末）
- 資料目錄：`/var/www/moodledata`
- 資料庫：類型 **MariaDB**、主機 `db`、名稱/帳號/密碼＝你 `.env` 的值
- 設管理員帳密、網站名稱（如「XX 社學習平臺」）

裝完就能登入後臺了。

---

## 步驟五：對應需求的後臺設定

### 1. 打卡（點擊即完成）
`網站管理 → 進階功能 → 啟用完成度追蹤 (enablecompletion)` 打勾。
之後每個課程/教材活動可設「檢視即標記完成」＝ 你要的「點擊打卡」。

### 2. 排行榜／XP（Level Up! 已裝）
`網站管理 → 外掛 → 有新外掛待安裝` → 按提示升級。
到課程裡新增 **Level Up! 區塊**，即有等級、XP、排行榜。全站排行榜在外掛設定開。

### 3. 三級權限（社員／幹部／社長）
`網站管理 → 使用者 → 權限 → 定義角色`：
- **社員** = 沿用 Student（可改中文名）
- **幹部** = 複製 Teacher，加開「建立/發布課程、看報表」能力
- **社長/副社長** = 複製 Manager，握「停權/提權、管理幹部」能力（`moodle/user:update`、角色指派等）

### 4. 課程審核（申請→審查）
課程的自我註冊設「需要審核」，或開 `啟用課程申請 (enablecourserequests)`，
幹部/社長在「待審核」清單核准或退回。

### 5. 課後派發
用 **作業(Assignment)** 交錄影/練習；互動題用 **H5P**（可設「看完影片才算完成」＝繞過外部影片追蹤難題）。

### 6. 排程 cron（Moodle 必需）
主機加一條每分鐘的 cron：
```bash
( crontab -l 2>/dev/null; echo "* * * * * cd ~/club_lms && docker compose exec -T moodle php admin/cli/cron.php >/dev/null 2>&1" ) | crontab -
```

## 之後：接網域 + HTTPS
1. 有網域後，DNS A 記錄指到公網 IP。
2. 前面加一層 Caddy（自動 Let's Encrypt 憑證）或走 Cloudflare。
3. 改 `moodle/config.php` 的 `$CFG->wwwroot` 成 `https://你的網域`（就這一行）。

需要時我再給你 Caddy 的 compose 片段。

## 常見狀況
- **打不開網頁**：先確認雲端 Security List 和主機 iptables 都放行 80。
- **arm64 映像**：本設定用官方 `php:8.3-apache`＋`mariadb:11`，都是多架構，ARM 原生。
- **改了外掛沒生效**：`docker compose restart moodle`，再進後臺跑升級。
